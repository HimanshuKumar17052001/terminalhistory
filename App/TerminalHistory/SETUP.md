# Xcode project setup

The Xcode project at `App/TerminalHistory.xcodeproj/` is generated by `scripts/generate-xcodeproj.rb` using the `xcodeproj` Ruby gem. It is committed to git so you can open it in Xcode immediately — no manual Xcode setup required.

## Regenerate

If you change the project layout, re-run:

```sh
gem install --user-install xcodeproj -v 1.25.0
ruby scripts/generate-xcodeproj.rb
```

This regenerates `App/TerminalHistory.xcodeproj` from scratch (including all file references, build phases, SwiftPM dependency on THCore, build settings, Info.plist reference, and entitlements reference).

## Build & run

In Xcode: open `App/TerminalHistory.xcodeproj` and press ⌘R.

From the command line:

```sh
xcodebuild -project App/TerminalHistory.xcodeproj -scheme TerminalHistory -configuration Debug build
```

## What the script configures

- Target: `TerminalHistory` (macOS app)
- Bundle ID: `com.himanshukumar17052001.TerminalHistory`
- Minimum deployment: macOS 13.0
- LSUIElement = YES (no Dock icon — menu bar only)
- App Sandbox: disabled (needed for `chsh`, `osascript`, writes to `~/Library`)
- Entitlements: `App/TerminalHistory/TerminalHistory.entitlements`
- Info.plist: `App/TerminalHistory/Info.plist`
- Local SwiftPM dependency: `THCore` (the repo root)
- All `.swift` files in `App/TerminalHistory/` are added to the target
- `Assets.xcassets` is added to the Resources build phase
- Shared scheme `TerminalHistory` is generated for command-line builds
