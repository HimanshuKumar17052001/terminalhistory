## [0.1.0] - 2026-06-26

Initial release. Personal OSS project, public at https://github.com/HimanshuKumar17052001/terminalhistory.

### Added

- **`THCore` SwiftPM library** (zero third-party deps):
  - `ULID` — 26-char Crockford-base32 identifier generator + parser (timestamp + 80 bits of randomness packed as a 5-bit stream).
  - `GzipCodec` — `Compression.framework` wrapper for transparent event payload compression.
  - `SessionStore` — SQLite (via `import SQLite3`) backed storage with FTS5 virtual table for full-text search over captured session text, plus automatic triggers keeping the FTS index in sync.
  - `Retention` — policy-driven pruning by age (default 90 days) and total DB size (default 5 GB), with pinned sessions exempt.
  - `Export` — writers for asciinema v2 (`.cast`) and plain text (`.txt`) formats.
  - `Search` — FTS5 query helpers with `snippet()` highlight rendering.
  - `Config` — JSON read/write of `~/.config/terminalhistory/config.json` (user shell, previous shell, default terminal, retention, theme).
  - `PTY` — `openpty(3)` pair allocator plus `forkpty`-based capturer that streams child stdout/stderr through to a callback while forwarding stdin from the parent.
  - `SessionRecorder` — buffers PTY bytes, gzips, writes events rows; captures `cwd` via a precmd-hook sidecar file in `TH_SESSION_DIR`.
  - `SessionReplayer` — produces a shell script that replays captured output (preserving ANSI escapes) and `exec`s the user shell in the original `cwd`.
  - `TerminalLauncher` — protocol + concrete launchers for Terminal.app (AppleScript), iTerm2 (AppleScript), Warp (URL scheme), and arbitrary custom commands.

- **`th` SwiftPM executable** (CLI doubling as the login shell wrapper):
  - `th setup` — detects current login shell, writes `~/.config/terminalhistory/config.json`, appends the cwd sidecar precmd hook to `~/.zshrc` / `~/.bashrc` / `~/.config/fish/config.fish`.
  - `th uninstall [--purge] [--yes]` — reverses setup: restores previous shell, removes the hook, deletes the binary and `.app`, optionally wipes data.
  - `th list [--limit N] [--day YYYY-MM-DD] [--pinned]` — recent sessions.
  - `th view <id>` — prints the replay script to stdout.
  - `th replay <id>` — opens the configured terminal (Terminal.app / iTerm2 / Warp / custom) and runs the replay followed by a fresh shell in the original cwd.
  - `th pin <id>` / `th unpin <id>`.
  - `th search <query> [--limit N]`.
  - `th export <id> --format cast|txt --out <path>`.
  - `th config get|set <key> [value]` for `default_terminal`, `user_shell`, `previous_shell`, `retention_max_days`, `retention_max_bytes`, `theme_icon`, `theme_accent`.
  - `th completions zsh|bash|fish` — emits completion scripts.
  - `th --shell <user-shell>` — login-shell mode: PTY-wraps the user shell, captures output and input to the session store, exports `TH_SESSION_DIR` and `TH_SESSION_ID` for the shell's precmd hook.

- **`TerminalHistory.app`** (Xcode project, generated programmatically via `scripts/generate-xcodeproj.rb` using the `xcodeproj` Ruby gem):
  - SwiftUI menu bar app with `LSUIElement = YES` (no Dock icon).
  - `MenuBarExtra` showing sessions grouped by day, with a Pinned section at the top.
  - Per-row star button for pin/unpin, click row to replay.
  - Search window reachable from the menu, with FTS5-backed query and snippet-highlighted results.
  - Preferences window with three tabs: General (default terminal, retention, login item toggle), Theme (icon variant + accent color), Storage (open data folder, run retention now).
  - Login item toggled via `SMAppService.mainApp`.
  - Themed menu bar label that honors the icon variant and accent color preference.

- **Install + uninstall**:
  - `install.sh` — builds `th` + `.app` from source, registers `~/.local/bin/th` in `/etc/shells` (sudo prompt once), runs `chsh`, opens the app.
  - `uninstall.sh` — shim that delegates to `th uninstall --yes`.

### Notes

- Build is fully reproducible from source: `git clone` → `./install.sh`.
- The Xcode project file (`App/TerminalHistory.xcodeproj`) is generated by `scripts/generate-xcodeproj.rb`. Running that script is idempotent and re-emits the project on demand.
- The library, CLI, and tests build cleanly with only Apple SDK frameworks (no SwiftPM dependencies).
